# OLS-3475: Rename Proposal to AgenticRun

Mechanical rename across the lightspeed-agentic-operator codebase. Aligns CRD kinds, API types, CLI commands, RBAC resources, audit events, OTEL spans, labels, and tests with the updated spec naming (`AgenticRun` / `AgenticRunApproval`).

## Approach

Big-bang scripted rename: ordered `sed`/`perl` replacements (longest strings first to avoid partial matches), file/directory renames via `git mv`, then `make manifests generate` to regenerate CRD YAML and deepcopy. One compile-fix cycle, one test run.

## Unchanged

- `DerivePhase()` function name (generic, not proposal-specific)
- Phase value `Proposed` (workflow state name, not type name)
- `SandboxStep` enum values (`Analysis`, `Execution`, `Verification`, `Escalation`)
- Result CRD kinds (`AnalysisResult`, `ExecutionResult`, `VerificationResult`, `EscalationResult`)
- SA naming pattern `ls-exec-{namespace}-{name}` (already generic)
- Finalizer value `agentic.openshift.io/execution-rbac-cleanup` (generic)

- All Go types renamed and `make build` passes
- `make manifests` regenerates CRD YAML with new Kind names
- CLI `oc-agentic run` works with all subcommands

- Audit events use `audit.agenticrun.*` prefix
- OTEL spans use `agenticrun.*` prefix
- Labels use `agentic.openshift.io/run`
- SA naming uses `ls-exec-{run-namespace}-{run-name}`
- All unit tests pass
- All e2e tests pass
